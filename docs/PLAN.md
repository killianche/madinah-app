Цель: к запуску MVP **веб-приложение замещает Telegram-бот @taubalessbot** без потери данных и с улучшенным UX. Работает на `app.madinah.ru`, хостится в РФ (152-ФЗ).

## Решения, которые уже приняты

| Вопрос | Решение |
|---|---|
| Стек | Next.js 15 + PostgreSQL 16 (не Supabase из-за 152-ФЗ) |
| Хостинг | TimeWeb Cloud (Москва) |
| Домен | `app.madinah.ru` (поддомен существующего) |
| Дизайн | Claude / Anthropic (parchment + terracotta + Serif) |
| Аутентификация | Lucia Auth + argon2id, сессии в БД |
| Партнёр-ревьюер | Нет |
| РКН уведомление | Владелец сам |
| Статусы урока | 4 (проведён, штрафной, отменён учителем, отменён учеником) |
| Баланс | может уходить в минус, показываем 0 если <0 для учителя |
| Пополнения | и учитель, и менеджер могут пополнять |
| Благотворительные | флаг `is_charity`, 3 режима в аналитике |
| Сохранение UUID из журнала | для учителей и учеников — ДА, для уроков — НЕТ (свежие) |
| Дедуп учеников при импорте | по UUID, затем по имени; остальное — ручной мерж менеджера |

## Риски и план митигации

| Риск | Митигация |
|---|---|
| Блокировка Telegram → потеря связи с учениками | Веб-приложение решает (есть в MVP) |
| Кто-то путает "Шамиль" и "Шамиль Инаев" | В карточке учителя показываем ФИО полностью, у "Шамиля" добавим уточнение после первого дня |
| 63 синтетических ученика могут оказаться дублями реальных | Админка "Объединить учеников" (V2) |
| Учителя не хотят менять привычки | Сохраняем 4 статуса один-в-один, UI-потоки как в боте, никаких новых полей в MVP |
| Падение VPS → простой | Managed Postgres с бэкапами; код в GitHub; Docker-образ собран и готов к передеплою |
| Утечка персональных данных | RLS в БД; audit_log пишет всё; argon2id; HTTPS обязателен |
