## Решения, которые уже приняты

См. **`docs/DECISIONS.md`** — там 12+ ADR с обоснованием.

Ключевые:
- Стек: Next.js 15 + Postgres 16 (не Supabase — 152-ФЗ)
- Хостинг: TimeWeb Cloud (Москва)
- Домен: `app.madinah.ru` (поддомен существующего)
- Auth: **свои сессии** (argon2id, SHA-256 токенов в БД; Lucia выкинули)
- 4 статуса урока: `conducted`, `penalty`, `cancelled_by_teacher`, `cancelled_by_student`
- Баланс: может уходить в минус; показываем 0, если <0 (для учителя)
- `is_charity` флаг на ученике, 3 режима в аналитике
- UUID учителей/учеников из журнала сохраняются; UUID уроков — свежие
- **Расписание** — шаблон «день недели + время», не список событий; повестка строится на лету
- **Перенос урока** — урок с `scheduled_date ≠ lesson_date`, один журнальный приход

## Риски и митигация

| Риск | Митигация |
|---|---|
| Блокировка Telegram → потеря канала | Веб-приложение решает |
| Путаница «Шамиль» и «Шамиль Инаев» | ФИО целиком в UI, архивные Шамили без user_id |
| 63 синтетических ученика из импорта | Экран «Объединить учеников» (V2) |
| Учителя не хотят менять привычки | 4 статуса как в боте; экран «Сегодня» — одна кнопка вместо формы |
| Падение VPS | Managed Postgres с бэкапами; Docker-образ в GitHub Container Registry (настроим); передеплой 5 минут |
| Утечка персональных данных | argon2id, сессии hashed в БД, audit_log, HTTPS обязателен, RLS в V2 |
| Минусовые балансы | Намеренно разрешены; менеджер видит в «Проблемных» |
