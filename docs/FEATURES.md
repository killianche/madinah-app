# Функционал Madinah App

Живой документ. Мы тут обсуждаем, добавляем, правим, отмечаем статус.
Правило: **функционал для учителя должен быть максимально простым и быстрым.**
Учитель ведёт 10+ учеников, каждый клик стоит времени.

**Легенда статуса:**
- ✅ готово — есть в коде, работает
- 🟡 частично — каркас есть, нужно допилить
- 🔲 запланировано — решили делать, ещё не начали
- 💬 обсудим — идея, хотим услышать «да/нет/по-другому»

## 🎓 Учитель

### Главный экран — «Сегодня» ✅

Открывается при входе. Показывает только то, что нужно **сейчас**:

1. **По графику** — ученики, у которых на сегодня запланирован слот по расписанию.
   Для каждого три кнопки без перехода между экранами:
   - **✓ Провёл** → статус `conducted`, баланс −1
   - **Отменён учеником** → `cancelled_by_student`, баланс не трогаем
   - **Штраф** → `penalty`, баланс −1 (ученик не пришёл без отмены)
   *Нажал → отметилось → строка «гасится». Всё. Ноль форм, ноль модалок.*

2. **Записано** — уже отмеченные за день уроки (подсветкой «вне графика», если были перенесены).

3. **Вне графика** — кнопка справа сверху. Открывает форму с чекбоксом «Перенос: урок был назначен на другой день» → учитель указывает плановую дату и фактическую. Всё сохраняется как один урок с двумя датами.

4. **Стрелки ‹ / ›** — листать дни. «сегодня» — быстрый возврат.

### Расписание ученика ✅

Показывается на карточке ученика. Это **регулярный шаблон**, не список будущих событий:

- «ученик А — пн 18:00, ср 18:00, пт 18:00»
- Учитель/менеджер добавляет/удаляет слоты одной кнопкой.
- Из слотов строится повестка «Сегодня».

### Карточка ученика ✅

- Баланс крупно.
- Действия: «Записать урок», «Пополнить баланс».
- Расписание (редактируемое).
- История уроков (фактическая дата + бейдж «план XX.XX», если был перенос).
- История пополнений.

### Пополнение баланса ✅

«+10 уроков, причина: оплатил 1500₽». Запись в `balance_topups` + audit_log.
*Физически деньги не принимаем — только учитываем, что баланс пополнен.*

### Чего у учителя **нет** (осознанно)

- Нет календаря-недели с drag-and-drop. Слишком сложно для телефона на ходу.
- Нет отметки «начать урок → закончить урок». Урок = строка в журнале по факту.
- Нет чата с учеником. Есть Telegram-username, можно кликнуть и написать в ТГ.

## 🧑‍💼 Менеджер

### Главный экран ✅

KPI школы:
- сколько активных учеников, сколько с балансом ≤ 0
- сколько уроков за неделю
- сколько учителей активно
- быстрая ссылка на «Проблемных»

### Проблемные ученики ✅

Композитный скор на основании:
- `days_since_last_lesson` × 0.3 — давно не было уроков
- `cancellations_last_30d` × 0.5 — часто отменяет
- `+2` если баланс < 5

Сортировка — сверху самые проблемные. Менеджер звонит/пишет в ТГ.

### Создать ученика ✅

Поля: ФИО, телефон, Telegram, учитель, стартовый баланс, `is_charity`.
После создания — переход на карточку ученика, где менеджер заводит расписание.

### 🔲 Запланировано

- **Редактирование ученика** (поменять учителя, сменить статус, перевести в paused/archived).
- **Список всех учеников** с поиском по имени/телефону/ТГ.
- **Экспорт уроков в CSV** (для отчётов и бухгалтерии).

## 🎓 Директор

### Главный экран 🟡 заготовка

Планируется:
- уроков по месяцам (график)
- топ-10 учителей по количеству уроков
- выручка за месяц (расчётная: платные уроки × средняя цена)
- % отмен по причинам

Для сложной аналитики — отдельный **Metabase** или **Redash**, подключённый к тому же Postgres в режиме read-only. В приложении — только «дашборд дня».

## 🔐 Администратор (ты)

### Создать пользователя ✅

Любая роль. Генерация временного пароля (12 символов, читаемый алфавит). Админ передаёт пароль учителю — при первом входе система попросит сменить.

### 🔲 Запланировано

- Список пользователей + деактивация.
- Сброс пароля (через временный токен + ссылка).
- Аудит — просмотр `audit_log` с фильтром по actor/entity.

## 📊 Данные и импорт

### Импорт из существующего журнала ✅

`scripts/import_from_csv.py`:
- читает CSV из текущего журнала (39 475 уроков)
- создаёт 69 учителей (31 активный + 38 архивных, их id сохраняются для истории)
- создаёт ~2017 учеников
- переносит все уроки со статусами и датами

Валидирован dry-run на реальном файле — 0 ошибок.

### 🔲 Запланировано

- **Плановые балансы после импорта** — сейчас балансы = 0, надо будет либо вручную, либо отдельной процедурой восстановить из истории (уже есть view `v_student_full` с `balance_calculated`).
- **Проверка целостности раз в сутки** — задача cron: сверять `students.balance` и `balance_calculated`.

## 🔔 Уведомления 🔲

**Идея:** бот в ТГ, который:
- пишет ученику за час до урока «у тебя урок в 18:00, учитель Ахмад»
- пишет учителю «завтра 5 уроков»
- пишет менеджеру если у ученика отменился 2-й урок за неделю

**Технически:** отдельный Telegram Bot API + таблица `notifications_queue` + cron каждые 5 минут.

Делать после пилота на 10 учителях.
